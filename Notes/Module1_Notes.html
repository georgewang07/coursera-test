Git Clone means you can clone everything in Github into the local repository

You can use "cd" to open any folder you are indicating

You can use "dir" to see all files in the current open repository

Nomal Process for push the content to the Remote/Github:

  1. git add . meaning put the content of the current repository to a place where it will be commited
  2. git commit -m "xxx" meaning commit on the added content with the message "xxx" which will come up in Github
  3. git push meaning push the committed content to the Remote

Notes: if there any issues about connection or proxy, try 2 methods then git push

  1. git config --global user.email "wanghao07@outlook.com" git config --global user.name "georgewang07"
  2. Or switch the Proxy: git config --global http.proxy http://127.0.0.1:7890 git config --global https.proxy http://127.0.0.1:7890

Lecture04: Basic HTML Doc Structure:

  1. There is always a starting line that is "" in HTML file to tells reads this is a HTML-format file
  2. After a starting line, "(html)xxx(html)" is mandatory to include everything into this file, like head, title, body, .etc
  3. "lang="EN" should be included into starting tag of "(html)"
  4. "(title)" means the browser tag name
  5. order: html - meta - head - title - body
  6. (meta) tag provides info about the web page to browsers and search engines
