Lecture13: Element, Class, ID, and grouping Selectors

  • Element Selector: just put Element before opening bracket;
  • Class Selector: place . before Class Name
  • ID Selector: place # before ID Name
  • grouping Selector: just put , bewtween each one

Lecture14: Combining Selectors

  • Element with Class: place . between Element Selector and Class Selector to indicate the declaration should be applied to Class of Element
  • Child Selector: place > between Element Selectors to indicate the later one that is the direct child of the former
  • Child's Child Selector: Place 2 >s between 3 elements, the declaration will applied to the 3rd one which is a direct child of the 2nd, which is a direct child of the 1st
  • Descendant Selectors: just place a space between Element Selectors to indicate the later one that is the any-level child of the former

Lecture19:Box Model

  • from the core to the outside: content > padding > border > margin
  • 4 directions from 1st to 4th: up > right > bottom > left
  • when box-sizing is border-box, the widthes of content, padding, border sum up to the "width", e.g. 300 = 200 (content width) + 2x30 (padding width) + 2x20(border width), which is common in many cases
  • if box-sizing: border-box is set, the width of the box, including padding and border, will be constrained to the specified width
  • content-box meaning the box width equals to content width
  • Margins don't define the width of the box. Just define how far other elements should be pushed away from it.
  • universal selector * { } style could change the default style of the elements/selectors

Lecture21: Positioning Elements by Floating

  • Floats are taken out of normal document flow
  • Use float to align elements and make them wrap around other content
  • Use clear to stop wrapping and start new content below floated elements; to resume normal document flow
